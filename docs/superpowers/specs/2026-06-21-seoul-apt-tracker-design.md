# 서울 아파트 가격 추적 시스템 — 설계 문서

**날짜:** 2026-06-21  
**목표:** 서울 500세대 이상 아파트 실거래가를 매월 자동 수집하고, 5년 월평균 추세 기반으로 다음달 가격을 프로젝션한다.

---

## 1. 아키텍처 개요

```
국토교통부 API ──→ Python 수집 스크립트 ──→ Google Sheets
  (실거래가)           (GitHub Actions)         (raw / monthly_avg / projection)
  (단지정보)            매월 1일 자동
```

**컴포넌트:**

| 컴포넌트 | 역할 |
|---|---|
| 국토교통부 실거래가 API | 서울 25개 구 월별 아파트 거래 데이터 소스 |
| 국토교통부 단지정보 API | 단지별 세대수 제공 (500세대+ 필터용) |
| Python 스크립트 | API 호출, 필터링, Sheets 업데이트 |
| GitHub Actions | 매월 1일 자동 실행 + 수동 트리거 |
| Google Sheets | 원본 적재, 집계, 프로젝션 결과 표시 |

---

## 2. 데이터 소스

두 API 모두 [data.go.kr](https://data.go.kr) 에서 무료 발급.

### 2-1. 아파트매매 실거래가 조회 서비스
- 조회 단위: 시군구코드 + 연월
- 주요 필드: 단지명, 법정동, 전용면적, 거래금액(만원), 건축년도, 층, 거래년월일

### 2-2. 공동주택 단지정보 서비스
- 단지별 세대수, 단지코드, 주소 제공
- 초기 1회 수집 → `data/complexes_500.csv` 로컬 캐시
- 이 목록으로 실거래가 데이터 교차 필터

**500세대 기준:** 단지정보 API에서 `세대수 >= 500` 단지만 추출. 서울 기준 약 1,000~1,500개 단지 예상.

## 3. Google Sheets 구조

### 탭 1: `raw`
원본 거래 데이터. 매월 append 방식.

| 컬럼 | 타입 | 예시 |
|---|---|---|
| 거래년월 | YYYY-MM | 2026-05 |
| 구 | 문자열 | 강남구 |
| 법정동 | 문자열 | 대치동 |
| 단지명 | 문자열 | 래미안대치팰리스 |
| 전용면적(㎡) | 숫자 | 84.99 |
| 면적구간 | 문자열 | 84㎡ |
| 거래금액(만원) | 숫자 | 285000 |
| 층 | 숫자 | 12 |
| 세대수 | 숫자 | 1608 |

예상 데이터량: 5년치 초기 적재 약 10~30만 행.

### 탭 2: `monthly_avg`
단지 × 면적구간 × 월 기준 평균 거래가. Python이 계산 후 기입.

- 면적구간 기준: 59㎡ 이하 / 60~84㎡ / 85~114㎡ / 115㎡ 이상
- 같은 단지·같은 면적구간의 월 거래들을 평균냄
- 거래가 없는 월은 공란 (보간 안 함)

### 탭 3: `projection`
5년 추세 기반 다음달 예상가.

| 컬럼 | 설명 |
|---|---|
| 단지명 | |
| 면적구간 | |
| 최근 실거래가(만원) | raw에서 가장 최근 거래 |
| 데이터 기간 | 분석에 사용한 월 범위 |
| 월평균 변동(만원) | 선형회귀 기울기 |
| 다음달 예상가(만원) | 최근가 + 월평균 변동 |
| 5년 CAGR(%) | 연환산 성장률 |
| 거래 횟수 | 분석 기간 내 거래 수 |

**프로젝션 방법:**
- 월평균가 시계열에 선형회귀 적용 → 기울기(만원/월) 추출
- `다음달 예상가 = 최근 거래 월의 평균가 + 기울기`
- 거래 횟수 5건 미만인 단지·면적구간은 프로젝션 제외 (신뢰도 부족)

## 5. GitHub Actions 워크플로우

```yaml
# .github/workflows/monthly.yml
on:
  schedule:
    - cron: '0 0 1 * *'   # 매월 1일 UTC 00:00 (한국 09:00)
  workflow_dispatch:        # 수동 실행 가능
```

**실행 순서:**
1. 전월 연월 계산 (예: 7월 1일 실행 → 6월 데이터 수집)
2. 서울 25개 구 코드 순회 → 실거래가 API 호출
3. `complexes_500.csv` 기준으로 500세대+ 단지만 필터
4. `raw` 탭에 신규 행 append
5. `monthly_avg` 탭 재계산
6. `projection` 탭 재계산

**필요한 GitHub Secrets:**

| Secret | 설명 |
|---|---|
| `MOLIT_API_KEY` | data.go.kr 발급 API 키 |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | Google Cloud 서비스 계정 JSON 키 전체 |
| `SPREADSHEET_ID` | Google Sheets URL 중간 ID |

## 6. 초기 셋업 절차 (한 번만)

1. **API 키 발급** — data.go.kr 회원가입 → "아파트매매 실거래가 조회 서비스" 활용 신청 (즉시 발급)
2. **Google 서비스 계정 생성** — Google Cloud Console → 서비스 계정 → JSON 키 다운로드 → 시트에 편집자 권한 부여
3. **GitHub 저장소 생성** — `the-trive` org 또는 개인 계정에 생성, Secrets 등록
4. **단지 목록 구축** — `python scripts/fetch_complexes.py` 1회 실행 → `complexes_500.csv` 생성
5. **과거 데이터 적재** — `python scripts/backfill.py --years 5` 1회 실행 → raw 탭에 60개월치 적재
6. **projection 초기 계산** — `python scripts/update_sheets.py` 실행
7. **이후 매월 자동**

---

## 7. 제약 및 주의사항

- **API 일일 호출 제한:** data.go.kr 기본 1,000건/일. 서울 25개 구 × 60개월 = 초기 적재 1,500건 → 2일에 나눠 실행 필요
- **거래 신고 지연:** 실거래 후 30~60일 신고 의무 → 전월 데이터도 다음달 초에 일부 누락 가능. 월 1일 수집 + 다음달 1일 보완 수집으로 대응
- **면적 편차:** 같은 "84㎡" 라도 84.01~84.99 등 소수점 차이 존재 → 구간 묶기로 처리
- **프로젝션 한계:** 선형회귀는 추세 연장이라 정책 변수(금리, 규제)는 반영 안 됨. 참고용 수치로 활용
